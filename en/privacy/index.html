Article 1 (Purpose of processing)

Run Pet is a mobile app for outdoor/indoor run logging and pet-motivation game features. The V1 release stores primary data on the user's device and does not synchronize it to a Run Pet server.

Article 2 (Personal information processed)

  • Precise GPS / route points — when a user starts an outdoor run, to record route, distance, pace and time. Active outdoor runs can continue location processing while backgrounded where OS permission/settings allow.
  • Running metrics and pedometer data — for records, indoor-running features, daily goals and in-app game progress.
  • User-selected photos — only when the user attaches a photo to a record.
  • Pet, egg, challenge, inventory, badge and settings data — local game and preference features.
  • Guest profile preferences and subscription entitlement flag — SharedPreferences (IAP not connected).

Processed on the device and not sent outside the device are different: V1 does not send these records to a Run Pet server, but map tiles can cause a network request to a configured map provider.

Article 3 (Retention period)

Records and game data are stored in device-local Drift/SQLite (schema v21), settings in SharedPreferences, and selected photos in the app's local document storage.

Raw GPS route geometry is stored for no more than 365 days and is then redacted while non-location running summaries may remain. Before an outdoor run, users separately accept the current location-based-service terms.

Article 4 (Provision to third parties)

V1 has no Run Pet remote account or server sync. When an operator configures a Google Maps API key, map tile requests may be subject to Google's privacy policy.

Article 5 (Outsourcing and SDK status)

  • Friend relations — feature shell only; backend off.
  • Ads / IDFA — Ad serving is disabled in current release builds. An ad SDK may be present in the app binary, but V1 release settings do not request ads or personalized advertising. No ATT.
  • Firebase Analytics / Remote Config — not initialized when project files are not configured.
  • Payment (StoreKit / Play Billing) — sales not connected.
  • Voice guidance — on-device TTS/native engine; no microphone recording or voice upload.

Article 6 (Destruction and user controls)

Users can stop/cancel a run, revoke OS location/photo/notification permissions, delete available records, withdraw location-terms consent to redact all saved route geometry, or remove the app. There is no V1 server account, so server-account deletion is NOT APPLICABLE.

Article 7 (App permissions)

  • Location (when-in-use / background) — outdoor run GPS
  • Motion / activity recognition — indoor pedometer
  • Photo library — attach photos to records
  • Notifications — Android outdoor foreground service when applicable

Article 8 (Minors)

Run Pet is for users aged 14 and over. The app stores only a local self-confirmation timestamp and does not collect date of birth or guardian information. Overseas availability does not remove mandatory protections that apply where a user lives.
